Protect employees, devices, and data with Zero Trust access, secure web gateway, and email security.

Modern enterprises need to secure a distributed workforce accessing applications from anywhere. Traditional perimeter-based security is no longer sufficient. Cloudflare One provides a complete SASE platform to protect users, devices, and data without sacrificing performance.

This guide walks you through the Cloudflare solutions available for implementing corporate security.

### Getting started with Access

<Details header="Set up Zero Trust access to an internal app">

**Connect your application with Tunnel:**

1. Install cloudflared on your server:
   ```sh
   curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64 -o cloudflared
   chmod +x cloudflared
   ```

2. Authenticate and create a tunnel:
   ```sh
   cloudflared tunnel login
   cloudflared tunnel create my-app-tunnel
   ```

3. Configure the tunnel:
   ```yaml
   tunnel: my-app-tunnel
   credentials-file: /path/to/credentials.json
   
   ingress:
     - hostname: app.example.com
       service: http://localhost:8080
     - service: http_status:404
   ```

4. Run the tunnel:
   ```sh
   cloudflared tunnel run my-app-tunnel
   ```

**Create an Access application:**

1. Go to **Zero Trust** > **Access** > **Applications**.
2. Select **Add an application** > **Self-hosted**.
3. Configure:
   - **Application name**: My Internal App
   - **Application domain**: `app.example.com`
4. Add an Access policy to control who can access.

</Details>

## Secure your company's internet access

Protect users from threats on the internet with DNS filtering and secure web gateway.

### Solutions

<CardGrid>

<LinkTitleCard title="Gateway" href="/cloudflare-one/policies/gateway/" icon="cloudflare-one">

Secure web gateway with DNS, HTTP, and network filtering.

</LinkTitleCard>

<LinkTitleCard title="DNS Filtering" href="/cloudflare-one/policies/gateway/dns-policies/" icon="cloudflare-one">

Block malicious domains and enforce acceptable use policies.

</LinkTitleCard>

<LinkTitleCard title="Browser Isolation" href="/cloudflare-one/policies/browser-isolation/" icon="cloudflare-one">

Isolate risky web content in a remote browser.

</LinkTitleCard>

</CardGrid>

### What you get

- **Threat protection** - Block malware, phishing, and command-and-control
- **Content filtering** - Enforce acceptable use policies
- **Data protection** - Prevent data exfiltration via web
- **Browser isolation** - Safely access risky sites without risk

### Getting started with Gateway

<Details header="Set up DNS filtering">

**Option 1: DNS-only (no client required)**

1. Go to **Zero Trust** > **Gateway** > **DNS Locations**.
2. Select **Add a location**.
3. Configure your network to use the assigned DNS addresses.

**Option 2: With WARP client (recommended)**

1. Deploy the WARP client to devices.
2. Go to **Zero Trust** > **Gateway** > **Policies** > **DNS**.
3. Create policies to block threats:

**Block security threats:**
- **Selector**: Security Categories
- **Operator**: in
- **Value**: All security threats
- **Action**: Block

**Block adult content:**
- **Selector**: Content Categories
- **Operator**: in
- **Value**: Adult Themes
- **Action**: Block

</Details>

<Details header="Configure HTTP inspection">

Enable HTTP inspection for deeper visibility:

1. Go to **Zero Trust** > **Settings** > **Network**.
2. Enable **TLS decryption**.
3. Deploy the Cloudflare root certificate to devices.
4. Create HTTP policies:

**Block file uploads to personal storage:**
- **Selector**: Application
- **Operator**: in
- **Value**: Personal Storage (Dropbox, Google Drive personal)
- **Action**: Block

**Isolate risky downloads:**
- **Selector**: Download File Types
- **Operator**: in
- **Value**: Executable files
- **Action**: Isolate

</Details>

For the complete guide, refer to [Gateway get started](/cloudflare-one/traffic-policies/).

## Stop emailphishing attacks

Protect your organization from email-based threats with advanced email security.

### Solutions

<CardGrid>

<LinkTitleCard title="Email Security" href="/email-security/" icon="email-security">

Stop phishing, BEC, and malware delivered via email.

</LinkTitleCard>

<LinkTitleCard title="DMARC Management" href="/dmarc-management/" icon="email-routing">

Prevent email spoofing and improve deliverability.

</LinkTitleCard>

</CardGrid>

### What you get

- **Phishing protection** - Block sophisticated phishing attacks
- **BEC prevention** - Detect business email compromise attempts
- **Malware scanning** - Stop malicious attachments
- **Brand protection** - Prevent domain spoofing with DMARC

### Getting started with Email Security

<Details header="Deploy email security">

**For Microsoft 365:**

1. Go to **Email Security** > **Settings**.
2. Select **Microsoft 365** as your email provider.
3. Follow the guided setup to:
   - Connect your Microsoft 365 tenant
   - Configure mail flow rules
   - Enable inline protection

**For Google Workspace:**

1. Go to **Email Security** > **Settings**.
2. Select **Google Workspace** as your email provider.
3. Follow the guided setup to configure routing.

**Configure detection settings:**

1. Go to **Email Security** > **Policies**.
2. Configure actions for different threat types:
   - **Malicious**: Quarantine
   - **Suspicious**: Add warning banner
   - **Spam**: Move to junk folder

</Details>

<Details header="Set up DMARC">

1. Go to **Email** > **DMARC Management**.
2. Add your domain and verify ownership.
3. Review DMARC reports to understand email sources.
4. Gradually enforce DMARC policy:
   - Start with `p=none` (monitor only)
   - Move to `p=quarantine` after review
   - Finally `p=reject` for full protection

</Details>

For the complete guide, refer to [Email Security get started](/email-security/deployment/).

## Prevent data loss

Protect sensitive data from exfiltration with DLP and CASB.

### Solutions

<CardGrid>

<LinkTitleCard title="Data Loss Prevention" href="/cloudflare-one/policies/data-loss-prevention/" icon="cloudflare-one">

Detect and block sensitive data in transit.

</LinkTitleCard>

<LinkTitleCard title="CASB" href="/cloudflare-one/applications/scan-apps/" icon="cloudflare-one">

Discover and secure SaaS applications.

</LinkTitleCard>

</CardGrid>

### What you get

- **Data detection** - Find sensitive data like PII, PCI, and PHI
- **Inline protection** - Block sensitive data from leaving
- **SaaS visibility** - Discover shadow IT and risky apps
- **Posture management** - Ensure SaaS apps are configured securely

### Getting started with DLP

<Details header="Configure DLP policies">

1. Go to **Zero Trust** > **Gateway** > **Policies** > **HTTP**.
2. Create a policy to detect sensitive data:

**Block credit card uploads:**
- **Selector**: DLP Profile
- **Operator**: in
- **Value**: Financial Information (Credit Cards)
- **Action**: Block

**Log PII transfers:**
- **Selector**: DLP Profile
- **Operator**: in
- **Value**: Personally Identifiable Information
- **Action**: Allow (with logging enabled)

**Custom patterns:**
Create custom DLP profiles for:
- Employee IDs
- Internal project codes
- Proprietary data patterns

</Details>

<Details header="Set up CASB">

1. Go to **Zero Trust** > **CASB** > **Integrations**.
2. Connect your SaaS applications:
   - Microsoft 365
   - Google Workspace
   - Salesforce
   - Slack

3. Review findings:
   - Misconfigurations
   - User risks
   - Data exposure

4. Create Gateway policies based on CASB findings.

</Details>

For the complete guide, refer to [DLP get started](/cloudflare-one/policies/data-loss-prevention/).

## Ensure device endpoint security

Verify device posture before granting access.

### Solutions

<CardGrid>

<LinkTitleCard title="Device Posture" href="/cloudflare-one/identity/devices/" icon="cloudflare-one">

Verify device security before granting access.

</LinkTitleCard>

<LinkTitleCard title="WARP Client" href="/cloudflare-one/connections/connect-devices/warp/" icon="cloudflare-one">

Secure connectivity and enforce policies on devices.

</LinkTitleCard>

</CardGrid>

### What you get

- **Posture checks** - Verify OS version, disk encryption, antivirus
- **Endpoint integration** - Connect with CrowdStrike, SentinelOne, etc.
- **Conditional access** - Grant access based on device health
- **Always-on protection** - WARP ensures traffic is always secured

### Getting started with device posture

<Details header="Configure device posture checks">

1. Go to **Zero Trust** > **Settings** > **WARP Client**.
2. Configure device enrollment settings.
3. Go to **Zero Trust** > **Settings** > **Device Posture**.
4. Add posture checks:

**Require disk encryption:**
- **Type**: Disk Encryption
- **Operating Systems**: All

**Require minimum OS version:**
- **Type**: OS Version
- **Operating System**: Windows
- **Version**: 10.0.19041 (Windows 10 2004)

**Require endpoint protection:**
- **Type**: Third-party endpoint security
- **Provider**: CrowdStrike
- **State**: Running

5. Add posture checks to Access policies:
   - Edit your Access application
   - Add a rule requiring the posture check

</Details>

For the complete guide, refer to [Device posture](/cloudflare-one/identity/devices/).

## Deployment approaches

### Start small

Begin with the highest-impact, lowest-friction deployments:

1. **DNS filtering** - No client required, immediate threat protection
2. **Access for one app** - Pilot Zero Trust with a single application
3. **Email security** - API deployment requires no mail flow changes

### Full deployment

Roll out comprehensive protection:

1. Deploy WARP client to all devices
2. Enable HTTP inspection with TLS decryption
3. Configure DLP policies for sensitive data
4. Integrate with endpoint security tools
5. Enable Browser Isolation for risky categories
