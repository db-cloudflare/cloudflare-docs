Build and deploy full-stack web applications globally with serverless compute, storage, and instant deployments.

Cloudflare's developer platform lets you build complete applications that run at the edge, close to your users. With Workers, Pages, and integrated storage products, you can deploy everything from simple websites to complex full-stack applications without managing infrastructure.

This guide walks you through the Cloudflare solutions available for building and deploying web applications.

---

## Deploy frontend applications

Ship static sites and full-stack applications with automatic builds, previews, and global distribution.

### Solutions

<CardGrid>

<LinkTitleCard title="Pages" href="/pages/" icon="pages">

Deploy frontend applications with Git integration, preview deployments, and global CDN.

</LinkTitleCard>

<LinkTitleCard title="Workers" href="/workers/" icon="workers">

Add server-side logic with Pages Functions or standalone Workers.

</LinkTitleCard>

</CardGrid>

### What you get

- **Git integration** - Automatic deployments from GitHub and GitLab
- **Preview deployments** - Every pull request gets a unique URL
- **Global CDN** - Static assets served from 300+ locations
- **Full-stack support** - Add server-side logic with Pages Functions

### Getting started with Pages

<Details header="Deploy a site with Pages">

1. Connect your Git repository:
   - Go to **Workers & Pages** in the dashboard
   - Select **Create application** > **Pages** > **Connect to Git**
   - Authorize access to your repository

2. Configure your build:
   - **Framework preset**: Select your framework (Next.js, Astro, etc.)
   - **Build command**: `npm run build`
   - **Build output directory**: Depends on framework (e.g., `dist`, `.next`, `build`)

3. Deploy:
   - Select **Save and Deploy**
   - Your site is live at `your-project.pages.dev`

4. Add a custom domain:
   - Go to your project > **Custom domains**
   - Add your domain and configure DNS

</Details>

For the complete guide, refer to [Pages get started](/pages/get-started/).

### Getting started with Workers

<Details header="Create and deploy a Worker">

1. Create a new project:
   ```sh
   npm create cloudflare@latest my-app
   cd my-app
   ```

2. Write your Worker code in `src/index.ts`:
   ```ts
   export default {
     async fetch(request: Request, env: Env): Promise<Response> {
       const url = new URL(request.url);
       
       if (url.pathname === "/api/hello") {
         return Response.json({ message: "Hello from the edge!" });
       }
       
       return new Response("Not found", { status: 404 });
     }
   };
   ```

3. Test locally:
   ```sh
   npm run dev
   ```

4. Deploy:
   ```sh
   npm run deploy
   ```

</Details>

- **Object storage** - Store files without egress costs
- **Real-time state** - Coordinate state with Durable Objects

### Getting started with D1

<Details header="Set up a D1 database">

1. Create a database:
   ```sh
   npx wrangler d1 create my-database
   ```

2. Add the binding to `wrangler.toml`:
   ```toml
   [[d1_databases]]
   binding = "DB"
   database_name = "my-database"
   database_id = "your-database-id"
   ```

3. Create a schema:
   ```sh
   npx wrangler d1 execute my-database --command "CREATE TABLE users (id INTEGER PRIMARY KEY, name TEXT, email TEXT)"
   ```

4. Query from your Worker:
   ```ts
   const users = await env.DB.prepare(
     "SELECT * FROM users WHERE id = ?"
   ).bind(userId).first();
   ```

</Details>

For the complete guide, refer to [D1 get started](/d1/get-started/).

### Real-time patterns

<Details header="Build a WebSocket chat with Durable Objects">

```ts
export class ChatRoom {
  sessions: Map<WebSocket, { name: string }> = new Map();

  async fetch(request: Request) {
    const url = new URL(request.url);
    
    if (url.pathname === "/websocket") {
      const pair = new WebSocketPair();
      const [client, server] = Object.values(pair);
      
      this.sessions.set(server, { name: "Anonymous" });
      server.accept();
      
      server.addEventListener("message", (event) => {
        const message = JSON.parse(event.data as string);
        this.broadcast(JSON.stringify({
          sender: this.sessions.get(server)?.name,
          text: message.text
        }));
      });
      
      server.addEventListener("close", () => {
        this.sessions.delete(server);
      });
      
      return new Response(null, { status: 101, webSocket: client });
    }
    
    return new Response("Not found", { status: 404 });
  }

  broadcast(message: string) {
    for (const [ws] of this.sessions) {
      ws.send(message);
    }
  }
}
```

</Details>

## Architecture patterns

### Full-stack application

Build a complete application with frontend and backend:

1. **Pages** hosts your frontend (React, Vue, Astro, etc.)
2. **Workers** (via Pages Functions) handles API routes
3. **D1** stores application data
4. **R2** stores user uploads and assets

### Real-time collaborative app

Build multiplayer or collaborative features:

1. **Durable Objects** coordinates state and WebSocket connections
2. **Workers** handles HTTP requests and routing
3. **KV** caches frequently accessed data
4. **Queues** processes background tasks

### Static site with dynamic features

Add interactivity to static content:

1. **Pages** serves static HTML/CSS/JS
2. **Workers** handles form submissions and API calls
3. **KV** stores form data and user preferences
4. **R2** stores uploaded files
