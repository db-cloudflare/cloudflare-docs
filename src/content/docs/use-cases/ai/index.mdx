AI applications present unique infrastructure challenges: unpredictable inference costs, latency-sensitive user experiences, and the need to work with multiple model providers. Cloudflare provides a complete platform for building AI applications that are fast, cost-effective, and globally distributed.

This guide walks you through the Cloudflare solutions available for building AI-powered applications.

### Getting started with Workers AI

<Details header="Set up Workers AI">

1. Create a new Worker project:
   ```sh
   npm create cloudflare@latest my-ai-app -- --template hello-world
   cd my-ai-app
   ```

2. Add the Workers AI binding to your `wrangler.toml`:
   ```toml
   [ai]
   binding = "AI"
   ```

3. Update your Worker code to run inference:
   ```js
   export default {
     async fetch(request, env) {
       const response = await env.AI.run("@cf/meta/llama-3.1-8b-instruct", {
         prompt: "What is Cloudflare Workers AI?"
       });
       return new Response(JSON.stringify(response));
     }
   };
   ```

4. Deploy your Worker:
   ```sh
   npx wrangler deploy
   ```

</Details>

For the complete guide, refer to [Workers AI get started](/workers-ai/get-started/).

- **Object storage** - Store documents, images, and generated content
- **Edge caching** - Cache frequent queries to reduce inference costs

### Getting started with Vectorize

<Details header="Set up Vectorize for RAG">

1. Create a Vectorize index:
   ```sh
   npx wrangler vectorize create my-index --dimensions=768 --metric=cosine
   ```

2. Add the Vectorize binding to your `wrangler.toml`:
   ```toml
   [[vectorize]]
   binding = "VECTORIZE"
   index_name = "my-index"
   ```

3. Insert vectors into your index:
   ```js
   const vectors = [
     { id: "doc1", values: [0.1, 0.2, ...], metadata: { text: "Document content" } }
   ];
   await env.VECTORIZE.upsert(vectors);
   ```

4. Query for similar vectors:
   ```js
   const queryVector = [0.1, 0.2, ...];
   const results = await env.VECTORIZE.query(queryVector, { topK: 5 });
   ```

</Details>

For the complete guide, refer to [Vectorize get started](/vectorize/get-started/).

### Example: Complete RAG application

<Details header="Build a RAG application">

Combine all the products to build a retrieval-augmented generation application:

```js
export default {
  async fetch(request, env) {
    const { query } = await request.json();

    // 1. Generate embedding for the query
    const embedding = await env.AI.run("@cf/baai/bge-base-en-v1.5", {
      text: query
    });

    // 2. Search for relevant documents
    const results = await env.VECTORIZE.query(embedding.data[0], {
      topK: 3,
      returnMetadata: true
    });

    // 3. Build context from retrieved documents
    const context = results.matches
      .map(match => match.metadata.text)
      .join("\n\n");

    // 4. Generate response with context
    const response = await env.AI.run("@cf/meta/llama-3.1-8b-instruct", {
      messages: [
        { role: "system", content: `Use this context to answer: ${context}` },
        { role: "user", content: query }
      ]
    });

    // 5. Store the interaction
    await env.DB.prepare(
      "INSERT INTO conversations (message, response) VALUES (?, ?)"
    ).bind(query, response.response).run();

    return Response.json({ answer: response.response });
  }
};
```

</Details>

## Architecture patterns

### Retrieval Augmented Generation (RAG)

Combine vector search with LLM inference to ground responses in your own data:

1. **Vectorize** stores embeddings of your knowledge base
2. **Workers** receives user queries and searches for relevant context
3. **Workers AI** or **AI Gateway** generates responses using retrieved context

### Multi-provider AI gateway

Use AI Gateway to route requests across providers while maintaining a single interface:

1. **AI Gateway** proxies requests to OpenAI, Anthropic, or Workers AI
2. Built-in caching reduces costs for repeated queries
3. Unified logging and analytics across all providers

### Real-time AI features

Deploy low-latency AI features directly at the edge:

1. **Workers** handles requests at the nearest Cloudflare location
2. **Workers AI** runs inference without round-trips to origin servers
3. **KV** or **D1** caches results and stores session state
