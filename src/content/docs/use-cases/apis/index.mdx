### Getting started with Workers

<Details header="Build a REST API with Workers">

1. Create a new Worker project:
   ```sh
   npm create cloudflare@latest my-api -- --template hello-world
   cd my-api
   ```

2. Create a REST API in `src/index.ts`:
   ```ts
   export default {
     async fetch(request: Request, env: Env): Promise<Response> {
       const url = new URL(request.url);
       
       // Route requests
       if (url.pathname === "/api/users" && request.method === "GET") {
         const users = await env.DB.prepare("SELECT * FROM users").all();
         return Response.json(users.results);
       }
       
       if (url.pathname === "/api/users" && request.method === "POST") {
         const body = await request.json();
         await env.DB.prepare(
           "INSERT INTO users (name, email) VALUES (?, ?)"
         ).bind(body.name, body.email).run();
         return Response.json({ success: true }, { status: 201 });
       }
       
       return new Response("Not Found", { status: 404 });
     }
   };
   ```

3. Deploy your API:
   ```sh
   npx wrangler deploy
   ```

</Details>

For the complete guide, refer to [Workers get started](/workers/get-started/).

### Getting started with API Shield

<Details header="Upload an OpenAPI schema">

1. Go to the [Cloudflare dashboard](https://dash.cloudflare.com/) and select your zone.
2. Go to **Security** > **API Shield**.
3. Select **Schema Validation** > **Upload schema**.
4. Upload your OpenAPI 3.x specification (JSON or YAML).

Example OpenAPI schema:
```yaml
openapi: 3.0.0
info:
  title: My API
  version: 1.0.0
paths:
  /api/users:
    get:
      responses:
        '200':
          description: List of users
    post:
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              required: [name, email]
              properties:
                name:
                  type: string
                email:
                  type: string
                  format: email
      responses:
        '201':
          description: User created
```

Cloudflare will automatically discover endpoints and create validation rules.

</Details>

For the complete guide, refer to [API Shield get started](/api-shield/get-started/).

### Configure rate limiting

<Details header="Set up API rate limiting">

1. Go to **Security** > **Application Security** > **Rate limiting rules**.
2. Select **Create rule**.
3. Configure the rule:
   - **Rule name**: `API rate limit`
   - **Expression**: `http.request.uri.path contains "/api/"`
   - **Rate**: `100 requests per 1 minute`
   - **Action**: `Block`

**Per-client rate limiting:**
- Use `IP Address` for per-IP limits
- Use `http.request.headers["authorization"]` for per-API-key limits

**Tiered rate limits:**
1. Warning: 80 requests/min → Log
2. Soft limit: 100 requests/min → Challenge
3. Hard limit: 200 requests/min → Block

</Details>

For the complete guide, refer to [Rate limiting rules](/waf/rate-limiting-rules/).

### Enable mTLS authentication

<Details header="Set up mTLS for API authentication">

**Create a client certificate:**

1. Go to **SSL/TLS** > **Client Certificates**.
2. Select **Create Certificate**.
3. Choose validity period and download certificate + key.

**Create an mTLS rule:**

<Render file="mtls-create-rule" product="api-shield" />

**Validate certificates in Workers:**

```ts
export default {
  async fetch(request: Request, env: Env): Promise<Response> {
    const clientCert = request.cf?.tlsClientAuth;
    
    if (!clientCert || clientCert.certVerified !== "SUCCESS") {
      return new Response("Client certificate required", { status: 403 });
    }
    
    // Certificate is valid, proceed with request
    return Response.json({
      authenticated: true,
      subject: clientCert.certSubjectDN
    });
  }
};
```

</Details>

### Authentication patterns

<Details header="JWT validation in Workers">

Validate JWTs at the edge:

```ts
import { verify } from "@tsndr/cloudflare-worker-jwt";

export default {
  async fetch(request: Request, env: Env): Promise<Response> {
    const authHeader = request.headers.get("Authorization");
    
    if (!authHeader?.startsWith("Bearer ")) {
      return new Response("Missing token", { status: 401 });
    }
    
    const token = authHeader.slice(7);
    
    try {
      const isValid = await verify(token, env.JWT_SECRET);
      if (!isValid) {
        return new Response("Invalid token", { status: 401 });
      }
      
      // Token is valid, proceed with request
      return handleRequest(request, env);
    } catch (error) {
      return new Response("Token validation failed", { status: 401 });
    }
  }
};
```

</Details>

<Details header="API key authentication">

Implement API key authentication:

```ts
export default {
  async fetch(request: Request, env: Env): Promise<Response> {
    const apiKey = request.headers.get("X-API-Key");
    
    if (!apiKey) {
      return new Response("API key required", { status: 401 });
    }
    
    // Look up API key in KV or D1
    const keyData = await env.API_KEYS.get(apiKey, "json");
    
    if (!keyData) {
      return new Response("Invalid API key", { status: 401 });
    }
    
    // Check rate limits, permissions, etc.
    if (keyData.rateLimit && await isRateLimited(apiKey, env)) {
      return new Response("Rate limit exceeded", { status: 429 });
    }
    
    // Proceed with authenticated request
    return handleRequest(request, env, keyData);
  }
};
```

</Details>

### Connect internal services

<Details header="Expose internal APIs with Tunnel">

1. Install cloudflared on your server:
   ```sh
   curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64 -o cloudflared
   chmod +x cloudflared
   ```

2. Authenticate and create a tunnel:
   ```sh
   cloudflared tunnel login
   cloudflared tunnel create my-api-tunnel
   ```

3. Configure the tunnel (`config.yml`):
   ```yaml
   tunnel: my-api-tunnel
   credentials-file: /path/to/credentials.json
   
   ingress:
     - hostname: api.example.com
       service: http://localhost:8080
     - service: http_status:404
   ```

4. Run the tunnel:
   ```sh
   cloudflared tunnel run my-api-tunnel
   ```

</Details>

### API observability

<Details header="Track API metrics with Analytics Engine">

```ts
export default {
  async fetch(request: Request, env: Env): Promise<Response> {
    const startTime = Date.now();
    const url = new URL(request.url);
    
    try {
      const response = await handleRequest(request, env);
      
      // Record successful request
      env.ANALYTICS.writeDataPoint({
        blobs: [url.pathname, request.method, response.status.toString()],
        doubles: [Date.now() - startTime],
        indexes: [url.pathname]
      });
      
      return response;
    } catch (error) {
      // Record error
      env.ANALYTICS.writeDataPoint({
        blobs: [url.pathname, request.method, "500", error.message],
        doubles: [Date.now() - startTime],
        indexes: [url.pathname]
      });
      
      return new Response("Internal error", { status: 500 });
    }
  }
};
```

Query metrics:
```sql
SELECT 
  blob1 as endpoint,
  blob2 as method,
  COUNT(*) as requests,
  AVG(double1) as avg_latency_ms,
  SUM(CASE WHEN blob3 >= '400' THEN 1 ELSE 0 END) as errors
FROM api_metrics
WHERE timestamp > now() - interval '1 hour'
GROUP BY endpoint, method
ORDER BY requests DESC
```

</Details>

## Architecture patterns

### Secure API gateway

Protect your APIs with defense in depth:

1. **API Shield** validates requests against your OpenAPI schema
2. **Rate Limiting** prevents abuse and DDoS attacks
3. **mTLS** authenticates known clients with certificates

### Edge-native APIs

Build APIs that run entirely on Cloudflare:

1. **Workers** handles request routing and business logic
2. **D1** or **KV** stores application data
3. **Queues** handles async processing and webhooks

### Microservices mesh

Connect and secure internal services:

1. **Cloudflare Tunnel** exposes services without public IPs
2. **Access** enforces identity-based policies between services
3. **Workers** acts as an API gateway for external consumers
