Deliver video, images, and rich media at scale with encoding, optimization, and global distribution.

Media-heavy applications require specialized infrastructure for encoding, storage, and delivery. Whether you are building a video platform, image-centric application, or content-rich website, Cloudflare provides end-to-end solutions for media workflows.

This guide walks you through the Cloudflare solutions available for building media and streaming applications.

### Getting started with Stream

<Details header="Upload and embed videos">

**Upload via Dashboard:**

1. Log in to the [Cloudflare dashboard](https://dash.cloudflare.com/) and select your account.
2. Go to **Stream** > **Videos**.
3. Select **Upload** and choose a video file.
4. Wait for encoding to complete.

**Upload via API:**

```sh
curl -X POST "https://api.cloudflare.com/client/v4/accounts/{account_id}/stream" \
  -H "Authorization: Bearer {api_token}" \
  -F "file=@/path/to/video.mp4"
```

**Embed the video:**

```html
<iframe
  src="https://customer-{code}.cloudflarestream.com/{video_id}/iframe"
  style="border: none; width: 100%; aspect-ratio: 16/9;"
  allow="accelerometer; gyroscope; autoplay; encrypted-media; picture-in-picture;"
  allowfullscreen="true"
></iframe>
```

**Use the Stream Player SDK:**

```html
<stream src="{video_id}" controls></stream>
<script data-cfasync="false" defer src="https://embed.cloudflarestream.com/embed/sdk.latest.js"></script>
```

</Details>

<Details header="Secure videos with signed URLs">

Enable signed URLs to prevent unauthorized access:

1. Go to **Stream** > **Videos** > select your video.
2. Select **Settings** > **Access Control**.
3. Enable **Require signed URLs**.

Generate tokens server-side:

```js
import { sign } from "@cloudflare/stream";

const signedUrl = await sign(videoId, {
  keyId: STREAM_KEY_ID,
  keyBase64: STREAM_KEY_BASE64,
  exp: Math.floor(Date.now() / 1000) + 3600, // 1 hour
});
```

</Details>

For the complete guide, refer to [Stream get started](/stream/get-started/).

Serve optimized images without pre-generating variants.

### Solutions

<CardGrid>

<LinkTitleCard title="Images" href="/images/" icon="images">

Transform, optimize, and deliver images at scale with on-the-fly resizing.

### What you get

- **On-the-fly transformation** - Resize, crop, and convert images via URL parameters
- **Modern formats** - Automatic WebP and AVIF conversion for supported browsers
- **Compression** - Reduce file sizes without visible quality loss
- **Responsive images** - Generate variants for different screen sizes

### Getting started with Images

<Details header="Upload and serve images">

**Upload via Dashboard:**

1. Go to **Images** > **Images**.
2. Select **Upload** and choose image files.

**Upload via API:**

```sh
curl -X POST "https://api.cloudflare.com/client/v4/accounts/{account_id}/images/v1" \
  -H "Authorization: Bearer {api_token}" \
  -F "file=@/path/to/image.jpg"
```

**Create variants:**

1. Go to **Images** > **Variants**.
2. Select **Add a variant**.
3. Configure settings (width, height, fit mode).

**Serve images:**

```
https://imagedelivery.net/{account_hash}/{image_id}/{variant_name}
```

</Details>

<Details header="Transform images on your domain">

Enable image transformations on your zone:

1. Go to **Speed** > **Optimization** > **Image Resizing**.
2. Enable **Image Resizing**.

Transform images via URL:

```
https://example.com/cdn-cgi/image/width=400,quality=80,format=auto/images/photo.jpg
```

**Supported parameters:**
- `width`, `height` - Resize dimensions
- `fit` - `scale-down`, `contain`, `cover`, `crop`, `pad`
- `quality` - 1-100 (default 85)
- `format` - `auto`, `webp`, `avif`, `json`
- `sharpen` - 0-10 sharpening amount

</Details>

For the complete guide, refer to [Images get started](/images/get-started/).

### What you get

- **Global distribution** - Automatic replication for fast access
- **Direct uploads** - Presigned URLs for client-side uploads

### Getting started with R2

<Details header="Create a bucket and upload files">

**Create a bucket:**

1. Go to **R2** > **Overview**.
2. Select **Create bucket**.
3. Enter a bucket name and select location hint.

**Upload via Dashboard:**

1. Go to your bucket > **Objects**.
2. Select **Upload** and choose files.

**Upload via S3 API:**

```sh
aws s3 cp video.mp4 s3://media-storage/ \
  --endpoint-url https://{account_id}.r2.cloudflarestorage.com
```

**Enable public access:**

1. Go to your bucket > **Settings**.
2. Under **Public access**, select **Allow Access**.
3. Connect a custom domain or use R2.dev subdomain.

</Details>

<Details header="Generate presigned URLs for uploads">

Allow users to upload directly to R2:

```js
import { S3Client, PutObjectCommand } from "@aws-sdk/client-s3";
import { getSignedUrl } from "@aws-sdk/s3-request-presigner";

const client = new S3Client({
  region: "auto",
  endpoint: `https://${ACCOUNT_ID}.r2.cloudflarestorage.com`,
  credentials: {
    accessKeyId: R2_ACCESS_KEY_ID,
    secretAccessKey: R2_SECRET_ACCESS_KEY,
  },
});

const command = new PutObjectCommand({
  Bucket: "media-storage",
  Key: "uploads/user-video.mp4",
});

const presignedUrl = await getSignedUrl(client, command, { expiresIn: 3600 });
```

</Details>

For the complete guide, refer to [R2 get started](/r2/get-started/).

### Cache configuration for media

<Details header="Optimize caching for media files">

**Configure Cache Rules for media:**

1. Go to **Caching** > **Cache Rules**.
2. Create rules for different media types:

**Video files (long TTL):**
- Match: `http.request.uri.path.extension in {"mp4" "webm" "m3u8" "ts"}`
- Cache TTL: 1 year
- Enable: Origin Cache Control

**Images (medium TTL):**
- Match: `http.request.uri.path.extension in {"jpg" "jpeg" "png" "webp" "avif"}`
- Cache TTL: 30 days
- Enable: Polish, WebP conversion

**Enable Tiered Cache:**

1. Go to **Caching** > **Tiered Cache**.
2. Enable **Smart Tiered Cache Topology**.

</Details>

### Content protection

<Details header="Implement token authentication">

**For Stream videos:**
Use signed URLs with expiration times (see Stage 1).

**For R2/origin content:**

Create a Worker to validate tokens:

```js
export default {
  async fetch(request, env) {
    const url = new URL(request.url);
    const token = url.searchParams.get("token");
    const expires = url.searchParams.get("expires");
    
    // Validate token
    if (!token || !expires) {
      return new Response("Unauthorized", { status: 401 });
    }
    
    if (Date.now() > parseInt(expires)) {
      return new Response("Token expired", { status: 403 });
    }
    
    const expectedToken = await generateToken(url.pathname, expires, env.SECRET);
    if (token !== expectedToken) {
      return new Response("Invalid token", { status: 403 });
    }
    
    // Serve the content
    return fetch(request);
  }
};
```

</Details>

## Architecture patterns

### Video platform

Build a complete video hosting and delivery solution:

1. **Stream** handles upload, encoding, and adaptive bitrate delivery
2. **Stream Live** enables live streaming with automatic recording
3. **Signed URLs** protect content with token authentication

### Image optimization pipeline

Serve optimized images without pre-generating variants:

1. **R2** stores original high-resolution images
2. **Images** transforms images on-the-fly based on URL parameters
3. **Workers** applies custom logic for format selection and caching

### User-generated content

Handle media uploads from users at scale:

1. **R2** receives uploads directly via presigned URLs
2. **Workers** validates and processes uploaded content
3. **Stream** or **Images** optimizes media for delivery
